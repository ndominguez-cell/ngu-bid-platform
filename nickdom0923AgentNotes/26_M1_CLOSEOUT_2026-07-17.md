## Advisor findings

### Fixed in `20260717120000_advisor_hardening.sql`

1. **Three REST-addressable SECURITY DEFINER helpers**
   - Moves `get_user_role()`, `is_workspace_member(uuid)`, and
     `shares_workspace_with(uuid)` from `public` to `private` with
     `ALTER FUNCTION ... SET SCHEMA`.
   - Moving the existing function objects preserves RLS dependencies, including
     policies outside the known core table list.
   - Replaces each definition with a locked empty search path and fully
     qualified table references.
   - Grants authenticated only the schema usage/function execution needed for
     policy evaluation. The `private` schema must remain excluded from exposed
     PostgREST schemas.

2. **Three `auth_rls_initplan` warnings on profiles**
   - Recreates `update_own`, `insert_own`, and `read_same_workspace` with
     `(select auth.uid())`.
   - Makes the private `shares_workspace_with` policy reference explicit.

3. **RLS enabled with zero policies**
   - Adds documented deny-all marker policies for anon/authenticated on
     `ai_rate_limits` and `auto_leads`; service-role access remains intentional.
   - `auto_leads` is live-schema drift not represented in existing migrations,
     so the SQL safely guards it with `to_regclass` for clean local rebuilds.

4. **Seventeen unindexed foreign keys**
   - `bid_activity(bid_id, user_id)`
   - `bids(company_id, contact_id)`
   - `contacts(company_id)`
   - `conversations(bid_id, contact_id)`
   - `documents(bid_id, uploaded_by, estimate_id)`
   - `estimates(bid_id, created_by)`
   - `proposals(bid_id, estimate_id, sent_by)`
   - `workspace_invitations(accepted_by, invited_by)`

5. **Invite ownership defense in depth**
   - Tightens the invitation role constraint to `admin`, `estimator`, or
     `viewer`; `owner` remains creation/backfill-only.
